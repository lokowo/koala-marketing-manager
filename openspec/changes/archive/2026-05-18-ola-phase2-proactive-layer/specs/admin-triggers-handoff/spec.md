## ADDED Requirements

### Requirement: Admin trigger management page
#### Scenario: View and manage triggers
- **WHEN** admin navigates to the triggers management page
- **THEN** all trigger rules are listed with page, message, enabled status, priority
- **AND** admin can edit, enable/disable triggers

### Requirement: Admin handoff queue page
#### Scenario: View pending handoffs
- **WHEN** admin navigates to the handoff queue page
- **THEN** pending handoff requests are listed with user info, reason, summary, timestamp
- **AND** admin can mark requests as handled
