审计日期: 2026-05-25

## 1. RP 模式的 System Prompt 完整内容

RP 功能有**两套独立的 system prompt**，分别用于不同场景：

### 1A. 聊天模式 RP Prompt (`app/lib/prompts/rp.ts`)

在 AI 对话页 `/koala/chat` 中，用户选择"📝 RP 助手"模式时使用。通过 `buildSystemPrompt('rp', extras)` 组装，最终 prompt = `Ola 人格 + 全局 System Prompt + RP_PROMPT + 额外上下文`。

RP_PROMPT 全文：

```
## 当前模式：Research Proposal 助手
你是一位资深的澳洲 PhD 研究计划（Research Proposal）写作顾问。

你的任务：
1. 帮助学生确定研究选题（结合教授的研究方向和学生的背景）
2. 指导 RP 的结构：研究背景、研究问题、文献综述、研究方法、时间线、参考文献
3. 审阅和修改学生写的 RP
4. 提供具体的写作建议和学术表达

## 工作流程
1. 先了解学生的背景和目标教授（如果有）
2. 讨论研究选题的可行性和创新性
3. 逐步引导完成 RP 各部分
4. 提供修改建议和学术表达改进

## RP 结构指导
- Title：简洁、具体、体现研究核心
- Background & Motivation（200-300 字）：领域现状 + 为什么需要这项研究
- Research Questions（2-3 个）：具体、可操作、有创新性
- Literature Review（简要）：关键文献 + 研究空白
- Methodology：适合研究问题的方法论
- Timeline（3-3.5 年分阶段）
- Expected Outcomes & Significance
- Key References（5-8 篇真实论文，有 DOI）

## 原则
- RP 长度通常 2000-3000 字（各校要求不同）
- 要体现学生对该领域的理解
- 研究问题要具体、可行、有创新性
- 方法论要适合研究问题
- 给出具体的、可操作的建议，而不是泛泛而谈
- 所有引用的论文必须真实，有 DOI
- 完成后提醒："这是 AI 辅助生成的初稿，建议与 KSA 学术顾问团队进一步打磨"
```

附加上下文（由 `route.ts` 在运行时注入）：
- 教授搜索工具指令（可调用 `searchProfessors` 从数据库检索真实教授）
- 对话阶段标记 `<stage>N</stage>`
- 快捷回复建议 `<suggestions>`
- 学术问题检测 RAG（如检测到学术意图，会并行搜索 Semantic Scholar + 知识库 + 教授库）

### 1B. 一键生成 RP Prompt (`app/api/user/research-proposal/generate/route.ts`)

在"我的文档"页面一键生成时使用，prompt 完全独立于聊天模式，内容如下：

```
你是一位资深的澳洲博士申请顾问，专精于帮助学生撰写针对特定导师的研究计划（Research Proposal）。

你的任务是根据学生背景和目标导师的研究方向，生成一份 2-3 页的澳洲式研究计划。

## 澳洲式研究计划特点
- 重事实与证据，非叙事风格
- 结构清晰，每段有明确目的
- 研究问题必须具体、可操作、有边界
- 方法论必须与问题匹配
- 必须展示对导师研究方向的深入理解

## 核心要求
1. 分析导师近期论文，推断其当前研究前沿和潜在兴趣
2. 将学生拟研究的问题框定为对导师方向的自然延伸或补充
3. 在 background 段落中适当引用导师的真实论文标题（用引号括起）
4. 如果导师无论文数据，基于 research_areas 推断方向
5. 所有内容输出为英文（学术标准）
6. 不要编造论文标题或数据——只引用提供的真实信息

## 输出格式（严格 JSON）
{
  "title": "...",
  "background": "...",
  "research_questions": "...",
  "methodology": "...",
  "significance": "...",
  "timeline": "..."
}
```

## 2. 用户输入什么信息后生成 RP

### 聊天模式（`/koala/chat` → RP 助手 tab）
- 自由对话形式，无固定输入字段
- 用户描述研究兴趣、粘贴 RP 草稿、或提问
- AI 引导式收集信息（背景、目标教授、研究方向）
- 快捷回复按钮：帮我确定研究选题 / 审阅我的 RP 草稿 / 怎么写文献综述 / 方法论怎么选

### 一键生成模式（`/koala/my-documents` 或 `/koala/my-applications`）
- **必需输入**: `professor_id`（从教授列表中选择目标导师）
- **可选输入**: `application_id`（关联申请）、`extra_context`（学生补充说明）
- **自动采集**（无需用户输入）:
  - 教授数据：名字、职位、大学、院系、研究方向、近期论文（最多 8 篇）、近期经费（最多 5 项）
  - 学生数据：从 `user_profiles` 自动读取（姓名、学校、专业、GPA、研究兴趣、研究经历、发表论文、教育经历、工作经历等）
  - AI 记忆：从 `memories` 表加载该用户的对话提取信息
- **前置条件**:
  - 必须登录
  - `profileCompleteness >= 20`（至少填写了教育背景和研究兴趣）
  - 用量限制：Free 用户 1 次/总计，Basic 3 次/总计，Pro/Enterprise 无限制

## 3. 生成的 RP 输出格式

### 聊天模式
- **纯文字 Markdown**，通过 AI 对话气泡展示
- 无结构化存储，无持久化（仅作为对话消息保存到 `ai_conversations` 表）
- 不支持直接导出 PDF

### 一键生成模式
- **结构化 JSON**，包含 6 个字段：
  - `title` — 研究计划标题（英文）
  - `background` — 文献回顾与研究空白（300-500 词）
  - `research_questions` — 研究问题与目标（200-300 词）
  - `methodology` — 研究方法（250-400 词）
  - `significance` — 研究意义与贡献（150-250 词）
  - `timeline` — 时间线（150-200 词）
- **保存到 `generated_documents` 表**，type='research_proposal'，status='draft'
- **前端渲染为可编辑卡片**（`ResearchProposalCard` 组件），支持：
  - 分段折叠/展开
  - 点击任意段落进入编辑模式
  - AI 润色（每段独立，调用 `/api/user/research-proposal/enhance-section`）
  - 保存修改（PATCH `/api/user/research-proposal/[id]`）
  - 状态切换（草稿 ↔ 定稿）
  - **PDF 导出**（调用 `/api/user/research-proposal/pdf`，使用 `@react-pdf/renderer` 生成 A4 格式学术论文样式 PDF）
  - 重新生成
  - 分享海报

## 4. 是否利用了教授匹配数据来定制 RP 内容

### 聊天模式 — 是（间接）
- RP 模式启用了 `searchProfessors` 工具（与 path/chat/write/interview 模式共享）
- AI 可以在对话中主动调用工具搜索真实教授数据
- 搜索返回教授的 research_areas、papers、matchScore、matchReasons
- 但**不会自动注入**教授数据，取决于 AI 是否判断需要调用工具

### 一键生成模式 — 是（深度集成）
- 必须指定 `professor_id`，从数据库加载：
  - 教授基本信息（name, university, faculty, position_title, research_areas, accepting_students）
  - 近期论文（从 `papers` 表 + `professors.latest_papers` 字段，去重，最多 8 篇，含 title/year/journal/citation_count）
  - 近期科研经费（从 `grants` 表 + `professors.latest_grants` 字段，去重，最多 5 项，含 title/funder/year）
- 所有教授数据**注入到 user message** 中，作为 AI 生成的核心依据
- System prompt 要求：
  - 分析导师近期论文推断研究前沿
  - 将学生方向框定为导师方向的自然延伸
  - 在 background 段引用导师真实论文标题
  - 不编造论文

## 5. 是否有保存和导出功能

### 保存
- **聊天模式**: 无独立保存。对话消息保存到 `ai_conversations` 表（与所有模式共用），但 RP 内容不作为结构化文档保存。
- **一键生成模式**: 自动保存到 `generated_documents` 表，字段包括：
  - `user_id`, `type='research_proposal'`, `professor_id`, `application_id`
  - `title`, `content`（JSON 对象含 6 段）, `status`（draft/final）, `credits_used`
  - 支持后续编辑保存（PATCH API）

### 导出
- **PDF 导出**: 通过 `POST /api/user/research-proposal/pdf`，生成学术论文格式 A4 PDF
  - 格式：Times-Roman 字体，学术标准排版
  - 包含：页眉（学生姓名 + 导师/大学信息）、标题居中、5 个编号分段、页脚（Generated by Koala PhD）、页码
  - 文件名：`Research_Proposal_{教授名}_{日期}.pdf`
- **列表查看**: 通过 `GET /api/user/research-proposal/list` 列出用户所有文档（含教授名和大学）

### AI 润色
- `POST /api/user/research-proposal/enhance-section`
- 支持对 5 个段落分别润色（background, research_questions, methodology, significance, timeline）
- 每个段落有专门的润色指令（加强引用、锐化研究问题、增加方法论细节等）
- 润色结果在前端以 preview 形式展示，用户可接受或放弃

## 6. 跟 P3-2 研究计划功能的关系

### 结论：当前代码中不存在独立的"P3-2 研究计划"功能。RP 功能是唯一的研究计划功能。

### 详细分析：

1. **项目历史中没有 P3-2 标记**: 搜索整个代码库和文档目录，未找到任何"P3-2"标记。PROJECT-STATE.md 中的 Phase 编号为：
   - P3 (Phase 8): Gmail 集成 + 推广工具 + 合规
   - P4-1 (Phase 9): 学术 CV 生成
   - P4-2 (Phase 9.3): 90 秒 Profile Capture + 邮件 Handoff

2. **RP 功能的两个实现路径是同一个功能的两种入口**:
   - **聊天入口**（`/koala/chat` → RP 助手 tab）: 交互式、引导式写作，使用 `app/lib/prompts/rp.ts` 的 prompt
   - **文档入口**（`/koala/my-documents` → 新建 → 研究计划）: 一键生成，使用 `app/api/user/research-proposal/generate/route.ts` 的独立 prompt

3. **共享基础设施**:
   - 两者最终都存储在 `generated_documents` 表（一键生成模式自动存，聊天模式不存）
   - 两者都可以利用教授数据库中的真实数据
   - 前端组件 `ResearchProposalCard` 同时在 `my-documents` 和 `my-applications` 页面使用

4. **Write 模式（写申请信）也涉及研究计划**: chat 页面的"✉️ 写申请信"模式描述为"定制申请信 · SOP · 研究计划"，但该模式使用的是 `WRITING_PROMPT`（独立于 `RP_PROMPT`），定位不同。

## 7. API 端点清单

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/ai/chat` (mode=rp) | POST | 聊天模式 RP 写作顾问 |
| `/api/user/research-proposal/generate` | POST | 一键生成结构化 RP |
| `/api/user/research-proposal/list` | GET | 列出用户所有文档 |
| `/api/user/research-proposal/[id]` | PATCH | 更新 RP 内容/标题/状态 |
| `/api/user/research-proposal/enhance-section` | POST | AI 润色单个段落 |
| `/api/user/research-proposal/pdf` | POST | 生成 PDF 下载 |
